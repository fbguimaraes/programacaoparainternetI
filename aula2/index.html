Passo a Passo: Configurando o Git e Enviando para o GitHub

1. Configurar nome de usuário e email

Após instalar o Git, abra o terminal (ou Git Bash no Windows) e configure o nome de usuário e o email. Eles são importantes porque aparecem nos registros de quem fez cada alteração.

git config --global user.name "Seu Nome" git config --global user.email "seuemail@example.com"

Você pode verificar se as configurações foram feitas corretamente com:

git config --list

2. Criar um novo repositório local

  1. Acesse a pasta onde deseja criar o repositório:
cd /caminho/para/sua/pasta
  1. Inicialize o repositório Git:
git init
  1. Crie um arquivo, por exemplo:
echo "# Meu Projeto" > README.md
  1. Adicione os arquivos ao Git:
git add .
  1. Faça o primeiro commit:
git commit -m "Primeiro commit"

3. Criar um repositório no GitHub

  1. Acesse o site https://github.com e faça login.
  2. Clique no botão "New" ou "Novo" para criar um novo repositório.
  3. Preencha o nome e as configurações desejadas e clique em "Create repository".

4. Conectar o repositório local ao GitHub

  1. Copie a URL do repositório que você acabou de criar no GitHub. Ela será parecida com:
https://github.com/SeuUsuario/NomeDoRepositorio.git
  1. No terminal, adicione o repositório remoto:
git remote add origin https://github.com/SeuUsuario/NomeDoRepositorio.git
  1. Envie os arquivos do repositório local para o GitHub:
git push -u origin master
Obs: Dependendo da versão do Git, o nome do branch pode ser main em vez de master. Se for o caso, use: git push -u origin main

5. Confirmar

  • Acesse novamente o seu repositório no GitHub e atualize a página.
  • Você verá os arquivos enviados.

Pronto!

Agora você já sabe como configurar o Git, criar um repositório e enviar seus arquivos para o GitHub.
